# Network Configuration Review Policy

## Purpose
Define SSC requirements for reviewing network and security configurations.

## Scope
All SSC network configurations, security groups, and perimeter controls.

## Compliance Status
SSC is not currently certified under SOC 2 or HIPAA. This policy defines readiness controls.

## Policy
- Review network configurations at least annually and after significant changes.
- Document justification for exposed services and ingress/egress rules.
- Track remediation of identified configuration risks.

## Standards and Procedures
- Maintain a network configuration review checklist.
- Document review outcomes and remediation actions.
- Align network changes with change management procedures.

## Roles and Responsibilities
- Operations: performs network configuration reviews.
- Security Lead: approves remediation plans.

## Review and Exceptions
- Review annually or upon significant changes.
- Exceptions require documented approval.
